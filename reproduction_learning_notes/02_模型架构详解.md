## 二、核心组件详解

### 2.1 RMSNorm (Root Mean Square Normalization)

```python
class RMSNorm(torch.nn.Module):
    def __init__(self, dim: int, eps: float = 1e-5):
        super().__init__()
        self.eps = eps
        self.weight = nn.Parameter(torch.ones(dim))

    def norm(self, x):
        return x * torch.rsqrt(x.pow(2).mean(-1, keepdim=True) + self.eps)

    def forward(self, x):
        return (self.weight * self.norm(x.float())).type_as(x)
```

**特点：**
- 相比LayerNorm，去掉了均值中心化操作
- 计算更快，训练更稳定
- 仅保留缩放因子γ

### 2.2 RoPE旋转位置编码 (Rotary Position Embedding)

```python
def precompute_freqs_cis(dim, end, rope_base, rope_scaling):
    freqs = 1.0 / (rope_base ** (torch.arange(0, dim, 2).float() / dim))
    t = torch.arange(end)
    freqs = torch.outer(t, freqs).float()
    freqs_cos = torch.cat([torch.cos(freqs), torch.cos(freqs)], dim=-1)
    freqs_sin = torch.cat([torch.sin(freqs), torch.sin(freqs)], dim=-1)
    return freqs_cos, freqs_sin
```

**核心思想：**
- 将位置信息编码到复数空间
- 通过旋转实现相对位置编码
- 支持YaRN外推，扩展上下文长度

**YaRN外推：**

### 2.3 GQA (Grouped Query Attention)

```python
class Attention(nn.Module):
    def __init__(self, config):
        self.num_key_value_heads = 4  # KV头数
        self.n_local_heads = 8        # Q头数
        self.n_rep = 2                # 重复系数
```

**优势：**
- 减少KV缓存，提升推理效率
- 平衡效果和计算开销
- 4个KV头，8个Q头，每2个Q头共享1个KV头

### 2.4 SwiGLU激活函数

```python
class FeedForward(nn.Module):
    def __init__(self, config):
        self.gate_proj = nn.Linear(config.hidden_size, intermediate_size, bias=False)
        self.down_proj = nn.Linear(intermediate_size, config.hidden_size, bias=False)
        self.up_proj = nn.Linear(config.hidden_size, intermediate_size, bias=False)
        self.act_fn = ACT2FN['silu']

    def forward(self, x):
        return self.down_proj(self.act_fn(self.gate_proj(x)) * self.up_proj(x))
```

**SwiGLU公式：**
```
SwiGLU(x) = (xW_gate ⊙ SiLU(xW_up)) W_down
```

**优势：**
- 比ReLU效果更好
- 门控机制增强表达能力

## 三、MoE架构 (Mixture of Experts)

### 3.1 MoE前馈层

```python
class MOEFeedForward(nn.Module):
    def __init__(self, config):
        self.num_experts = 4           # 专家数量
        self.num_experts_per_tok = 1   # 每个token选择的专家数
        self.gate = nn.Linear(hidden_size, num_experts, bias=False)
        self.experts = nn.ModuleList([FeedForward(config) for _ in range(num_experts)])
```

**路由机制：**
1. Gate网络计算每个token对各专家的权重
2. 选择Top-K专家
3. 加权聚合专家输出

### 4.3 学习率调度

```python
def get_lr(it, total_lr, learning_rate):
    # Warmup + Cosine Decay
    warmup_steps = total_steps * 0.1
    if it < warmup_steps:
        return learning_rate * it / warmup_steps
    else:
        progress = (it - warmup_steps) / (total_steps - warmup_steps)
        return learning_rate * 0.5 * (1 + math.cos(math.pi * progress))
```

## 五、LoRA实现

### 5.1 LoRA核心思想

在原有权重矩阵旁引入低秩增量分支：
```
W' = W + ΔW = W + BA
其中 B ∈ R^(d×r), A ∈ R^(r×d), r << d
```

### 5.2 代码实现

```python
class LoRALinear(nn.Module):
    def __init__(self, in_features, out_features, r=8):
        self.linear = nn.Linear(in_features, out_features, bias=False)
        self.lora_a = nn.Linear(in_features, r, bias=False)
        self.lora_b = nn.Linear(r, out_features, bias=False)

    def forward(self, x):
        return self.linear(x) + self.lora_b(self.lora_a(x)) * self.scaling
```

**参数效率：**
- LoRA参数量: 0.393M
- 占总参数: 0.61%
- 极大降低微调成本

## 六、关键设计决策

### 6.1 为什么选择dim=768, n_layers=8？

根据MobileLLM研究：
- **深而窄** vs **宽而浅**：深度比宽度更重要
- 768维度不至于过小导致模式崩溃
- 8层训练更快，同时保持效果

### 6.2 为什么词表大小6400？

- 压缩embedding层和输出层参数占比
- 适合小模型体积约束
- 实际使用中足够稳定可用

### 6.3 GQA vs MHA vs MQA

| 方法 | KV头数 | 特点 |
|------|--------|------|
| MHA | = Q头数 | 效果最好，显存最大 |
| GQA | < Q头数 | 平衡选择 |
| MQA | 1 | 推理最快，效果损失 |

MiniMind选择GQA (4 KV heads, 8 Q heads)

## 七、学习心得

1. **架构设计是权衡艺术**：没有完美的架构，只有适合的架构
2. **小模型有大学问**：64M参数的模型也能学到有用的语言模式
3. **从零实现的价值**：理解底层原理比调用API更重要
4. **工程细节决定成败**：混合精度、梯度累积、学习率调度都影响最终效果
