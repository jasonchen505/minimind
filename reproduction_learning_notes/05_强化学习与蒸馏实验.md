# 强化学习与知识蒸馏实验笔记

## 一、DPO (Direct Preference Optimization) 实验

### 1.1 DPO原理

DPO是一种直接优化偏好的强化学习方法，无需训练奖励模型。

**核心思想：**
- 输入：(chosen, rejected) 偏好对
- 目标：增加chosen概率，降低rejected概率
- 约束：与参考模型保持KL散度

**损失函数：**
```
L_DPO = -log σ(β · (log π(y_w|x)/π_ref(y_w|x) - log π(y_l|x)/π_ref(y_l|x)))
```

其中：
- π: 当前策略模型
- π_ref: 参考模型（冻结的SFT模型）
- y_w: 更好的回答 (chosen)
- y_l: 较差的回答 (rejected)
- β: 温度参数

### 1.2 数据格式

```json
{
  "chosen": [
    {"content": "Q", "role": "user"}, 
    {"content": "good answer", "role": "assistant"}
  ], 
  "rejected": [
    {"content": "Q", "role": "user"}, 
    {"content": "bad answer", "role": "assistant"}
  ]
}
```

### 1.3 训练过程观察

```
Epoch:[1/1](50/2146), loss: 0.6905, dpo_loss: 0.6905
Epoch:[1/1](500/2146), loss: 0.5419, dpo_loss: 0.5419
Epoch:[1/1](1000/2146), loss: 0.6919, dpo_loss: 0.6919
Epoch:[1/1](2146/2146), loss: 0.5947, dpo_loss: 0.5947
```

**关键观察：**
1. Loss从0.69降到0.59
2. 训练相对稳定
3. 需要两个模型：策略模型和参考模型

### 1.4 DPO vs PPO

| 特性 | DPO | PPO |
|------|-----|-----|
| 奖励模型 | 不需要 | 需要 |
| 训练稳定性 | 更稳定 | 较不稳定 |
| 实现复杂度 | 简单 | 复杂 |
| 效果 | 接近 | 接近 |

## 二、知识蒸馏实验

### 2.1 蒸馏原理

知识蒸馏分为两类：
1. **黑盒蒸馏**：学习教师模型的输出
2. **白盒蒸馏**：学习教师模型的输出分布

**白盒蒸馏损失：**
```
L = α · CE(y_true, p_student) + (1-α) · T² · KL(p_teacher^T || p_student^T)
```

其中：
- T: 温度参数
- α: CE损失权重
- KL: KL散度

### 2.2 训练配置

```python
# 蒸馏参数
teacher_model: full_sft (教师模型)
student_model: full_sft (学生模型初始化)
alpha = 0.5      # CE损失权重
temperature = 1.5  # 蒸馏温度
```

### 2.3 训练过程观察

```
Epoch:[1/1](50/113215), loss: 1.1580, ce: 2.3138, distill: 0.0022
Epoch:[1/1](500/113215), loss: 1.2018, ce: 2.3998, distill: 0.0038
Epoch:[1/1](2700/113215), loss: 1.2400, ce: 2.4721, distill: 0.0078
```

**关键观察：**
1. 总Loss约1.2
2. CE损失约2.4（主导）
3. 蒸馏损失约0.007（较小）
4. 蒸馏损失逐渐增加（学习教师分布）

### 2.4 蒸馏的作用

1. **知识迁移**：从大模型到小模型
2. **性能提升**：小模型获得更好效果
3. **训练稳定**：比直接训练更稳定

## 三、PPO/GRPO实验（未完成）

### 3.1 PPO原理

PPO是经典的策略梯度算法：
1. Actor生成回答
2. Reward Model打分
3. 更新Actor策略
4. KL散度约束

**需要的组件：**
- Actor模型（策略模型）
- Reward Model（奖励模型）
- Reference Model（参考模型）

### 3.2 GRPO改进

GRPO (Group Relative Policy Optimization)：
- 组内相对排序
- 无需绝对奖励值
- 更稳定的训练

### 3.3 未完成原因

GRPO需要下载internlm2-1_8b-reward奖励模型，当前环境无法访问HuggingFace。

1. **DPO的优势**：无需训练奖励模型，直接优化偏好
2. **训练稳定性**：比PPO更稳定
3. **效果接近**：与PPO效果相当

2. **性能提升**：小模型获得更好效果

1. **每个阶段都有价值**：预训练→SFT→RLHF→蒸馏
