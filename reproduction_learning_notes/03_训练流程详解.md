# MiniMind训练流程详解

## 一、预训练阶段 (Pretrain)

### 1.1 预训练目标

预训练的核心目标是让模型学会**高质量地词语接龙**：
- 输入"秦始皇" → 生成"是中国历史上的第一位皇帝"
- 学习事实知识、语言模式和上下文统计关系

### 1.2 数据格式

```json
{"text": "如何才能摆脱拖延症？治愈拖延症并不容易，但以下建议可能有所帮助。"}
{"text": "Transformer 通过自注意力机制建模上下文关系，是现代大语言模型的重要基础结构。"}
```

**特点：**
- 纯文本，无标注
- 覆盖多种领域
- 中英混合

### 1.3 训练配置

```python
# 预训练参数
epochs = 1-2
batch_size = 16-32
learning_rate = 5e-4
max_seq_len = 340  # tokens长度
```

### 1.4 训练过程观察

**Loss变化：**
- 初始: ~7.7 (随机猜测)
- 中期: ~5.0 (学会基本语法)
- 后期: ~3.8 (掌握一定知识)

**关键观察：**
1. 前1000步下降最快
2. 之后缓慢收敛
3. 学习率warmup很重要

## 二、有监督微调 (SFT)

### 2.1 SFT目标

将预训练模型转换为**对话助手**：
- 理解user/assistant/system角色
- 遵循指令生成回答
- 适应多轮对话格式

### 2.2 数据格式

```json
{
    "conversations": [
        {"role": "user", "content": "你好"},
        {"role": "assistant", "content": "你好！"},
        {"role": "user", "content": "再见"},
        {"role": "assistant", "content": "再见！"}
    ]
}
```

**特点：**
- 多轮对话
- 角色分明
- 包含Tool Call数据

### 2.3 训练配置

```python
# SFT参数
epochs = 1
batch_size = 16
learning_rate = 1e-5  # 比预训练小很多
max_seq_len = 512
```

### 2.4 Loss Mask机制

```python
def generate_labels(input_ids):
    labels = [-100] * len(input_ids)
    # 只在assistant回答位置计算loss
    # user输入位置mask掉
    return labels
```

**关键点：**
- 只对assistant回答计算loss
- user输入不参与梯度计算
- 防止模型学习"复读"用户输入

## 三、LoRA微调

### 3.1 LoRA原理

**核心思想：**
```
W' = W + ΔW = W + BA
其中 B ∈ R^(d×r), A ∈ R^(r×d), r << d
```

**优势：**
- 参数量减少99%+
- 训练速度快
- 保留通用能力

### 3.2 适用场景

1. **垂域适配**：医疗、法律、金融等专业领域
2. **自我认知**：让模型记住自己的身份
3. **风格调整**：改变回答风格

### 3.3 训练配置

```python
# LoRA参数
r = 8  # 秩
alpha = 16  # 缩放因子
target_modules = ['q_proj', 'k_proj', 'v_proj', 'o_proj']
```

## 四、强化学习 (RLHF/RLAIF)

### 4.1 DPO (Direct Preference Optimization)

**核心思想：**
- 直接优化偏好，无需训练奖励模型
- 输入：(chosen, rejected) 对
- 目标：增加chosen概率，降低rejected概率

**损失函数：**
```
L_DPO = -log σ(β log π(y_w|x)/π_ref(y_w|x) - β log π(y_l|x)/π_ref(y_l|x))
```

### 4.2 PPO (Proximal Policy Optimization)

**流程：**
1. Actor生成回答
2. Reward Model打分
3. 更新Actor策略
4. KL散度约束

### 4.3 GRPO (Group Relative Policy Optimization)

**改进：**
- 组内相对排序
- 无需绝对奖励值
- 更稳定的训练

## 五、训练技巧总结

### 5.1 学习率调度

```python
# Warmup + Cosine Decay
warmup_steps = total_steps * 0.1
if step < warmup_steps:
    lr = base_lr * step / warmup_steps
else:
    progress = (step - warmup_steps) / (total_steps - warmup_steps)
    lr = base_lr * 0.5 * (1 + cos(π * progress))
```

### 5.2 梯度裁剪

```python
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

**作用：**
- 防止梯度爆炸
- 稳定训练

### 5.3 混合精度训练

```python
# bfloat16更稳定
dtype = torch.bfloat16
with torch.cuda.amp.autocast(dtype=dtype):
    loss = model(input_ids, labels=labels).loss
```

### 5.4 梯度累积

```python
loss = loss / accumulation_steps
loss.backward()

if step % accumulation_steps == 0:
    optimizer.step()
    optimizer.zero_grad()
```

## 六、训练监控

### 6.1 关键指标

1. **Loss**: 训练损失，越低越好
2. **Learning Rate**: 学习率变化
3. **Gradient Norm**: 梯度范数，监控稳定性
4. **Tokens/s**: 训练速度

### 6.2 常见问题

**问题1：Loss不下降**
- 检查学习率是否合适
- 检查数据格式是否正确
- 检查梯度是否消失

**问题2：Loss震荡**
- 降低学习率
- 增加batch size
- 检查数据质量

**问题3：过拟合**
- 增加数据量
- 使用正则化
- 早停

## 八、学习心得

### 8.1 训练是一门艺术

1. **数据质量 > 数据数量**：高质量数据更重要
2. **学习率是关键**：太大震荡，太小收敛慢
3. **监控很重要**：及时发现问题

### 8.2 从预训练到对话

1. **预训练**：学习语言基础
2. **SFT**：学习对话格式
3. **RLHF**：学习人类偏好

### 8.3 实践出真知

- 纸上得来终觉浅
- 动手实验才能真正理解
- 失败是最好的老师
